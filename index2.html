Normal HTML Elements vs. Semantic HTML Elements

Normal HTML Elements: Think of them like basic building blocks. They're versatile but don't tell much about their purpose.

Semantic HTML Elements: These are descriptive, like labels on building blocks. They give a clearer picture of what's inside. For example, instead of just saying "block," you can label it as "header" or "footer."

Disparity between Closing Tags and Self-closing Tags

Closing Tags: They're like the last piece in a puzzle, marking the end of something. For example, when you finish a paragraph, you need to say, "That's all for this paragraph" by using a closing tag like </p>.

Self-closing Tags: These are for things that don't need an ending because they're self-contained, like an image or a line break. It's like saying "Here's an image" without needing to say "End of image" afterward.

Inline Elements vs. Block Elements

Inline Elements: They're like words within a sentence, seamlessly fitting in without causing a break. They're great for small things like emphasizing a word or creating a link.

Block Elements: These are like big building blocks in a wall, each starting a new line and taking up the full width. They're perfect for creating distinct sections like paragraphs or headers.
